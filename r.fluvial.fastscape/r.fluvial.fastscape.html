DESCRIPTION

r.fluvial.fastscape computes D8 flow routing on a DEM and, optionally, evolves that landscape forward in time under rock uplift and detachment-limited fluvial incision.

By default (nsteps=0) it performs flow routing only: a depression-fill, the D8 flow-direction raster, and flow accumulation — a fast O(n) drainage-routing solver (the Braun & Willett ordered-stack machinery) usable as a routing front-end to r.fluvial.channelheads, alongside r.watershed, r.fill.dir, and r.richdem.

With nsteps > 0 it also integrates the stream-power model 

dz/dt = U - K A^m S^n
where z is elevation, U rock-uplift rate, A drainage area, S channel slope, and K, m, n the stream-power coefficient and exponents, using the implicit, stack-ordered FastScape algorithm of Braun & Willett (2013), which is unconditionally stable: arbitrarily large timesteps do not cause the solution to blow up.

Each timestep: rock uplift is added to the interior, flow is routed on a depression-filled surface (D8), drainage area and the ordered node stack are computed, and the incision term is solved implicitly from downstream to upstream (closed-form for n = 1, Newton–Raphson otherwise). The solver shares its Braun & Willett flow-routing machinery with r.fluvial.channelheads.

NOTES

The four domain edges are fixed as base level by default (flag -b disables this). Cells are assumed square; for non-square regions the north–south resolution is used and a warning is issued. Per-cell K and U rasters may be supplied with k_map and uplift_map.

At steady state the model reproduces the classic channel slope–area relation S = (U/K)1/n A-m/n.

The routing outputs describe the final surface: the input DEM when nsteps=0 (route only), or the evolved landscape when nsteps > 0. The direction raster uses the r.watershed encoding (1–8 counter-clockwise from NE; 0 = pit/outlet) — the canonical FastScape/LSDFlowInfo routing, which feeds straight into r.fluvial.channelheads's direction input or r.stream.distance, keeping routing internally consistent across the toolkit. accumulation is the number of upstream cells (as in r.watershed), and filled is the depression-filled DEM used for routing.

EXAMPLE

# flow routing only (default nsteps=0): drainage directions + accumulation,
# then feed the directions into channel-head extraction
r.fluvial.fastscape input=dem direction=draindir accumulation=accum
r.fluvial.channelheads method=dreich elevation=dem direction=draindir points=heads

# landscape evolution (nsteps > 0), then route the evolved surface
r.fluvial.fastscape input=dem output=dem_evolved direction=draindir \
    k=1e-5 m=0.5 n=1 uplift=1e-3 dt=1000 nsteps=200
r.fluvial.channelheads method=dreich elevation=dem_evolved direction=draindir \
    points=heads
