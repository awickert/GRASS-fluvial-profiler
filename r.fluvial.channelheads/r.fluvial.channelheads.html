DESCRIPTION

r.fluvial.channelheads locates the upstream limit of the channel network. It offers two methods, which find two distinct (but related) quantities:
  • method=dreich — DrEICH morphological channel heads (Clubb et al., 2014), a faithful Python port of the LSDTopoTools chi–elevation algorithm. Channel heads are the cells where the long profile becomes linear in the integral coordinate χ, downstream of the persistently non-linear hillslope. Needs only the DEM.
  • method=slope_area (default) — the colluvial-to-fluvial transition, the downslope limit of colluvial hollows, from the slope–area break. This is a process boundary that lies downslope of the DrEICH head.

method=dreich (DrEICH channel heads)

On the depression-filled DEM the module: routes D8 flow and accumulates area; extracts an over-dense source set at a small area threshold; fits second-order-polynomial tangential curvature (window_radius) and walks downstream from each source, flagging a valley where the curvature stays above tan_curv_threshold for n_connecting_nodes connected cells; and for each valley builds the hilltop→junction χ profile (a_0, m_over_n) and splits it into an upslope hillslope and a downslope channel segment by a regression test (R2 of the channel vs. Durbin–Watson of the hillslope, segments at least min_segment_length nodes). The furthest-upstream head per branch is kept. The algorithm lives in rivernetworkx.dreich; the GRASS module is a thin wrapper.

Routing. By default the D8 flow routing is computed internally with the same FastScape-style algorithm that r.fluvial.fastscape uses (the canonical DrEICH pathway). You may instead supply an external drainage-direction raster with direction (the r.watershed encoding: 1–8 counter-clockwise from NE, negative off-region). r.watershed -s (single-flow-direction, steepest descent) reproduces the internal routing closely; the default multiple-flow-direction mode does not, so use -s if you route with r.watershed. Tangential curvature is always computed internally on the filled DEM (it cannot be supplied externally without changing the result).

Outputs. Any combination of three may be requested (at least one is required): channel-head points (output); the fluvial network — the channel heads and everything downstream of them — as vector lines (network); and/or that same network as a raster stream map (network_raster). The vector network traces the D8 flow from each head to its outlet and splits it into links at confluences, in v.stream.network format: every link carries its upstream/downstream endpoints (x1,y1,x2,y2) and the category of the single downstream link (tostream, with 0 marking a link that exits the map). The result is therefore already a converging directed graph — no separate v.stream.network run is required — ready for the graph tools in rivernetworkx. The raster form is a CELL map whose cell values are the link categories (matching the vector network), NULL off the network, for downstream GRASS modules that consume a stream raster.

method=slope_area (colluvial-to-fluvial transition)

In log–log slope–area space, hillslopes (and colluvial hollows) have a slope that is roughly independent of drainage area, while fluvial channels are concave (S ≈ k A−θ). The rollover between the two regimes marks where the landscape crosses from hillslope/colluvial to fluvial process dominance.

This method:

  1. reads a deliberately over-extracted stream network (so it reaches upslope past the hollows), sampling elevation and flow accumulation along it;
  2. computes per-segment, flowline-smoothed channel slope and pairs it with drainage area;
  3. fits a continuous, constrained broken-stick to the slope–area cloud (a flat hillslope/colluvial limb and a power-law fluvial limb joined at a knot), whose knot is the colluvial-to-fluvial transition drainage area A*;
  4. places a transition point where each channel's drainage area first crosses A* (the hollow gives way to a fluvial channel). Flow accumulation already integrates the upstream area, so no network topology is needed.

The analysis lives in the shared NetworkX-based rivernetworkx library; this module is a thin GRASS wrapper over it.

NOTES

This maps a process boundary — where fluvial processes begin to dominate over hillslope/colluvial ones — which is a complete and reproducible quantity, and is the appropriate upstream limit of the fluvial domain (e.g. for seeding a fluvial long-profile model). It is not a field channel head: a morphological channel head, such as that of Clubb et al. (2014)'s DrEICH method, lies upslope of this transition (at smaller drainage area), within the colluvial hollow. The two are related but distinct; the distance between them is the extent of the colluvial channel reach.

streams must be over-extracted: run r.stream.extract with a small threshold so the network over-shoots upslope past the hollows, placing the slope–area rollover within the extracted network. The accumulation raster (e.g. from r.watershed) must cover the whole network.

Per-cell slope is noisy; set window to a flowline smoothing distance (map units; a few tens of metres for metre-scale LiDAR) so the slope is smoothed along each segment before the break is fit. min_slope drops near-flat valley cells (numerically zero slope) that would otherwise bias the fit, and max_area can exclude large downstream trunk reaches.

The fitted A* is a single, basin-wide transition area, which assumes a uniform process domain. In strongly heterogeneous terrain the transition is better described by a slope–area threshold that varies with local slope; a per-flowline variant is a possible future refinement.

If the region does not contain the full basin (a clipped sub-area), channels entering from off-map carry negative r.watershed accumulation at their heads; pass -c to treat that as a boundary artifact rather than off-map inflow.

EXAMPLE

# DrEICH morphological channel heads (needs only the DEM)
r.fluvial.channelheads method=dreich elevation=dem output=channel_heads \
    threshold=100 a_0=1000 m_over_n=0.525 window_radius=7

# ...and also emit the downstream fluvial network, as vector lines and a raster
r.fluvial.channelheads method=dreich elevation=dem output=channel_heads \
    network=channel_network network_raster=channel_network_rast \
    threshold=100 window_radius=7
# the 'tostream' column links each segment to its downstream segment (0 = exits map)
v.db.select map=channel_network columns=cat,tostream

# Route with r.watershed instead of the internal D8 (use -s for steepest descent)
r.watershed -s elevation=dem drainage=draindir
r.fluvial.channelheads method=dreich elevation=dem direction=draindir \
    network=channel_network threshold=100 window_radius=7

# Colluvial-to-fluvial transition from the slope-area break
r.watershed -s elevation=dem accumulation=accum drainage=draindir
r.stream.extract elevation=dem accumulation=accum threshold=100 \
    stream_vector=streams_dense stream_raster=streams_rast
r.fluvial.channelheads method=slope_area elevation=dem accumulation=accum \
    streams=streams_dense output=hollow_transition window=30 min_slope=0.0001
