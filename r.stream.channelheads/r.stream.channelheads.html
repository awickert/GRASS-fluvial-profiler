DESCRIPTION

r.stream.channelheads locates fluvial channel heads — the points where channels become fluvial — from the slope–area break. In log–log slope–area space, hillslopes have a slope that is roughly independent of drainage area, while fluvial channels are concave (S ≈ k A−θ). The transition between the two regimes — the rollover — marks where channels initiate.

The module:

  1. reads a deliberately over-extracted stream network (so it reaches upslope past the real heads), sampling elevation and flow accumulation along it;
  2. computes per-segment, flowline-smoothed channel slope and pairs it with drainage area;
  3. fits a continuous, constrained broken-stick to the slope–area cloud (a flat hillslope limb and a power-law fluvial limb joined at a knot), whose knot is the fluvial-initiation drainage area A*;
  4. places a channel head where each channel's drainage area first crosses A* (becomes fluvial). Flow accumulation already integrates the upstream area, so no network topology is needed.

The analysis lives in the shared NetworkX-based rivernetworkx library; this module is a thin GRASS wrapper over it.

NOTES

This finds fluvial-initiation heads, a complete and reproducible quantity: it marks every fluvial channel head in the network, so it will return more points than a sparse, field-mapped head set. It is not the field-equivalent channel head of Clubb et al. (2014)'s DrEICH method, which sits somewhat further upslope (at smaller drainage area), in the colluvial–fluvial transition; a DrEICH-equivalent detector is planned separately.

streams must be over-extracted: run r.stream.extract with a small threshold so the network over-shoots upslope, placing the slope–area rollover within the extracted network. The accumulation raster (e.g. from r.watershed) must cover the whole network.

Per-cell slope is noisy; set window to a flowline smoothing distance (map units; a few tens of metres for metre-scale LiDAR) so the slope is smoothed along each segment before the break is fit. min_slope drops near-flat valley cells (numerically zero slope) that would otherwise bias the fit, and max_area can exclude large downstream trunk reaches.

If the region does not contain the full basin (a clipped sub-area), channels entering from off-map carry negative r.watershed accumulation at their heads; pass -c to treat that as a boundary artifact rather than off-map inflow.

EXAMPLE

# Flow routing and a deliberately dense (over-extracted) network
r.watershed -s elevation=dem accumulation=accum drainage=draindir
r.stream.extract elevation=dem accumulation=accum threshold=100 \
    stream_vector=streams_dense stream_raster=streams_rast

# Fluvial channel heads from the slope-area break
r.stream.channelheads elevation=dem accumulation=accum streams=streams_dense \
    output=channel_heads window=30 min_slope=0.0001
