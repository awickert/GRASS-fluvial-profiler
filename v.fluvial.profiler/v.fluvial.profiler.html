DESCRIPTION

v.fluvial.profiler builds and exports plots and data sets of river long profiles, slope-accumulation (e.g., slope-area), and more for topographic analysis. Starting from a chosen segment (cat), it walks either downstream to the map outlet (a single channel) or upstream through every tributary that drains to it. Elevation, flow accumulation, and slope rasters are sampled along the channel; the profile can be densified to a target spacing (dx_target) and smoothed with a moving-average window (window). Results can be plotted, written as a text table (outfile), and exported as a NetworkX node-link JSON sub-network (json).

NOTES

v.fluvial.profiler requires its input streams vector to have been linked by v.fluvial.network first (it reads the tostream column to walk the network); it will stop with an error otherwise. The graph construction, raster sampling, and JSON I/O are provided by the rivernetworkx Python package, which must be importable in the GRASS Python environment (pip install -e .).

When accumulation is supplied, an incomplete catchment (contributing area extending off the map) is detected: GRASS r.watershed marks such cells with negative flow accumulation, and if that reaches a channel head the module stops with an error (off-map upstream inflow is not yet supported). If the region truly contains the full basin (e.g. a drainage divide on the region edge), pass the -c flag to treat those negatives as a boundary artifact and proceed. Accumulation with no data (NULL) under a channel cell is always an error.
